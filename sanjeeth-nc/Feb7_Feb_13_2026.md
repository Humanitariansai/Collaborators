---

## Overview
This week focused on **building interactive analytics dashboards using React** to surface BimRide's forecasting outputs, booking trends, and operational KPIs in a visual, actionable format.  
The goal was to design a modular, component-driven dashboard architecture that connects to BimRide's existing Snowflake data layer through FastAPI endpoints, rendering real-time and historical analytics for the operations and marketing teams.

React was chosen for its **component reusability, rich ecosystem of charting libraries, and ability to build highly interactive UIs** that go beyond static reporting — enabling drill-downs, filters, and live-updating panels that traditional BI tools struggle to customize.

## Dummy Project Work (Step by Step)

### Part 1: React Dashboard Foundation

#### Step 1: Project Scaffolding & Tooling
- Initialized the dashboard project using **Vite + React + TypeScript** for fast builds and type safety.  
- Configured **ESLint and Prettier** for consistent code formatting across the dashboard codebase.  
- Set up **TailwindCSS** for utility-first styling, enabling rapid layout iteration without custom CSS overhead.  
- Established folder structure following **feature-based organization**: `/components`, `/hooks`, `/services`, `/pages`, `/utils`.  
- Integrated **React Router** for multi-page navigation between Bookings, Drivers, Revenue, and Forecasting views.

#### Step 2: Data Visualization Library Evaluation
| **Library** | **Chart Types** | **Customization** | **Performance** | **Selected For** |
|---|---|---|---|---|
| **Recharts** | Line, Bar, Area, Pie | High (React-native components) | Good for < 5K points | Booking trend charts |
| **Nivo** | Heatmaps, Choropleth, Calendar | Very High (theming system) | Moderate | Demand heatmaps by zone |
| **React-Leaflet** | Maps, Markers, Polygons | High (plugin ecosystem) | Good | Driver location & zone maps |
| **TanStack Table** | Data grids, sorting, filtering | Very High | Excellent (virtualized) | Tabular ride & driver data |

- Selected **Recharts** as the primary charting library for its declarative API and seamless React integration.  
- Integrated **React-Leaflet** for geographic visualizations of driver positions and demand zones across Barbados.  
- Chose **TanStack Table** for sortable, filterable data grids displaying ride history and driver performance metrics.

#### Step 3: State Management & Data Fetching
- Implemented **TanStack Query (React Query)** for server state management — handling API caching, background refetching, and stale data invalidation.  
- Designed **custom hooks** (`useBookingMetrics`, `useDriverStats`, `useRevenueData`, `useForecast`) to encapsulate data fetching logic and keep components lean.  
- Configured **polling intervals** (30-second refresh for live panels, 5-minute refresh for historical views) to balance real-time updates with API load.  
- Built a centralized **API service layer** using Axios with interceptors for authentication headers and error handling.

### Part 3: Production Readiness & UX Polish

#### Step 7: Responsive Design & Accessibility
- Implemented **responsive grid layouts** using TailwindCSS breakpoints — dashboard adapts from desktop (3-column) to tablet (2-column) to mobile (single-column).  
- Added **keyboard navigation** and ARIA labels to all interactive elements for accessibility compliance.  
- Built a **dark mode toggle** using Tailwind's dark variant classes and React context for persisted user preference.  
- Optimized chart rendering with **React.memo** and **useMemo** to prevent unnecessary re-renders on large datasets.

#### Step 8: Error Handling & Loading States
- Implemented **skeleton loaders** for all panels during data fetch, providing visual feedback without layout shift.  
- Built an **error boundary component** with retry functionality wrapping each dashboard section independently — one panel failure doesn't crash the entire dashboard.  
- Added **empty state illustrations** for scenarios with no data (new zones, off-hours with zero bookings).  
- Configured **toast notifications** for API errors and successful data exports using a lightweight notification library.

## Challenges Faced
1. **Chart Performance with Large Datasets** – Rendering 90 days of hourly booking data (~2,160 points) caused jank in Recharts; resolved by downsampling to daily aggregation for longer ranges and using virtualized rendering.  
2. **WebSocket Reconnection** – Driver location WebSocket connections dropped intermittently; implemented exponential backoff reconnection with state reconciliation on reconnect.  
3. **Time Zone Handling** – Barbados (AST, UTC-4) vs. server UTC timestamps created display inconsistencies; standardized all frontend display to AST using date-fns-tz.  
4. **Forecast Overlay Alignment** – Aligning predicted vs. actual booking data on the same chart axis required careful date normalization when forecast data extended beyond available actuals.  
5. **Component Re-render Cascades** – Deeply nested state updates caused cascading re-renders across unrelated panels; resolved by splitting React Query cache keys per panel and memoizing expensive computations.

## BimRide Application (Ride-Hailing)

### Operational Decision Support
The React dashboard gives BimRide's operations team a **single-pane-of-glass view** into real-time bookings, driver availability, and demand forecasts. Instead of switching between spreadsheets and Snowflake queries, dispatchers can monitor live demand, identify underserved zones on the map, and trigger driver reallocation — all from one interface.

### Forecast-Driven Planning
The forecast overlay panel enables the team to **visually compare predictions against actuals**, building trust in the forecasting models built in previous weeks. The what-if simulator empowers non-technical staff to explore demand scenarios without writing code, democratizing data-driven decision making.

### Stakeholder Reporting
Built-in **CSV/PNG export** and responsive design allow managers and investors to access analytics from any device. The revenue waterfall chart provides instant visibility into which service lines drive the most revenue, supporting strategic decisions about fleet composition and service expansion.
